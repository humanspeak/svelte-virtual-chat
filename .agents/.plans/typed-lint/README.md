## Adaptation notes (differences from the svelte-markdown source plan)

- **Smaller blast radius**: only 7 unit `.test.ts` files are unlinted here; the
  20 Playwright `.spec.ts` files don't match the ignore glob and are already
  linted (untyped). svelte-markdown had ~150 dark files.
- **Tooling contract now matches svelte-markdown** (adopted 2026-07-09, same
  session): CLAUDE.md's "Code Style & Linting" section makes Trunk the source
  of truth ‚Äî never run `pnpm lint`/`pnpm format`/`npx prettier`/`npx eslint`,
  and never use `eslint-disable` comments (Trunk `trunk-ignore` inline ignores
  only, with a rationale). The repo's one pre-existing `eslint-disable-next-line`
  in `chatMeasurement.svelte.ts` was converted to `trunk-ignore` as part of
  that adoption; `src/lib/types.ts:6` is the other exemplar.
- **Type-project shape differs**: `.svelte-kit/tsconfig.json` already includes
  `tests/**`, so specs/fixtures get type info for free; but
  `playwright.config.ts` and `vitest.setup.ts` sit outside the project and need
  `projectService.allowDefaultProject`.
- **Added a lint-time STOP** (~3 min budget) because the husky pre-commit hook
  runs trunk on every commit in this repo ‚Äî a slow typed-lint config directly
  taxes every future commit.
